# My Crafty Stash — Mac + iPad (.NET MAUI)

Cross-platform companion to the [My Crafty Stash](https://github.com/TequilaJosh/MyCraftyStash)
Windows desktop app, built with .NET MAUI so the whole app is C# + XAML +
CommunityToolkit.Mvvm — the same stack as the desktop app.

**Targets:** iPadOS (`net10.0-ios`), macOS (`net10.0-maccatalyst`), and a
Windows head (`net10.0-windows...`) used for day-to-day development — you can
run and click the app on a PC while the identical code ships to Apple
platforms.

**Phase 1 (current): read-only cloud stash browser.** Sign in with an `mcs_`
API key generated by the Windows app (Settings › Cloud Sync › API Keys) and
browse the inventory the desktop synced to the cloud. Roadmap:
[docs/PORT-PLAN.md](docs/PORT-PLAN.md).

## Building

```bash
dotnet workload install maui
cd MyCraftyStash
# Run on Windows (development):
dotnet build -t:Run -f net10.0-windows10.0.19041.0
# Compile-check the Apple targets (requires macOS + Xcode):
dotnet build -f net10.0-ios
dotnet build -f net10.0-maccatalyst
```

No Mac handy? Every push to `main` compile-checks the iOS and Mac Catalyst
targets on GitHub's macOS runners (`.github/workflows/build.yml`). That
workflow is the source of truth for "does it build on Apple" until real
hardware arrives; the Windows head is how we see and use the app meanwhile.

## Architecture notes

- Auth: the same `mcs_` API keys as the desktop, sent as `X-Api-Key`, stored
  via MAUI `SecureStorage` (Keychain on Apple platforms, DPAPI on Windows).
- `Models/` mirrors the cloud API JSON; dates stay ISO-8601 strings.
- Server dependency: `/api/items` + `/api/stash/summary` must accept
  `X-Api-Key` — that change is parked on the website repo's `apple-api-key`
  branch pending testing (only `/api/whoami` accepts keys on live today).
